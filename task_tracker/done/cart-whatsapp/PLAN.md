# Корзина + оформление заказа через WhatsApp (cart-whatsapp)

> Статус: done
> Дата: 2026-06-10
> Тип: фича

## Цель

Дать покупателю набрать корзину на витрине и оформить заказ: заказ записывается в БД
(страховка + задел под Фазу 3), затем открывается WhatsApp заказчицы с готовым текстом
заказа (wa.me click-to-chat) — покупателю остаётся нажать «Отправить». Заказчица видит
все заказы в админке. Это MVP-вариант оформления из go-live-чеклиста (§4) — без оплаты
и без учёта остатков (наличие подтверждается в переписке).

## Видение результата

- На карточке товара: выбор цвета (есть) + **размера** (новое) → «В корзину».
- В Header — иконка корзины со счётчиком; клик открывает drawer справа: позиции
  (фото, цвет, размер, qty ±, удалить), итого, пометка «Доставка и самовывоз по Алматы».
- «Оформить заказ в WhatsApp» → server action пишет `Order`+`OrderItem` в БД (цены
  перепроверяются по БД) → экран «Заказ №N принят»: кнопка WhatsApp (wa.me с готовым
  текстом), на десктопе QR-код для телефона, копирование текста, телефон.
- Корзина НЕ чистится автоматически; повторное оформление неизменённой корзины
  переиспользует тот же заказ №N (дедуп по слепку корзины в localStorage).
- Админка: раздел «Заказы» — список с составом, суммой и сменой статуса вручную.
- `coming_soon`-товары: кнопка «Узнать о наличии» → wa.me с вопросом (вместо alert).
- Админка: в форме товара — поля «Ссылка Kaspi» / «Ссылка Ozon» (кнопки на карточке
  уже рендерятся через существующий `MarketplaceLinks`).
- География на карточке: «Алматы — заказ через корзину · Казахстан — Kaspi ·
  другие страны — Ozon» (сегменты МП — только при заполненных ссылках).
- Номер WhatsApp — поле в `site_settings`, редактируется в «Настройках сайта»;
  ссылка WhatsApp появляется в футере.

## Out-of-scope (сознательно НЕ делаем)

- Проверка/резерв остатков (`stockQty`/`reservedQty` не трогаем) — Фаза 2,
  после неё резерв при оформлении добавится одной операцией в `createOrder`.
- Онлайн-оплата — Фаза 4.
- Синхронизация с маркетплейсами, `MarketplaceListing` — Фаза 5.
- Telegram-пуш заказчице о новом заказе — бэклог.
- Форма контактов покупателя — решено НЕ делать (номер виден в WhatsApp-чате).
- WhatsApp Business API — отклонён (см. обсуждение: дорого, рвёт диалог с покупателем).
- Автооткрытие WhatsApp без клика (window.open при ответе action) — не делаем,
  popup-блокеры; всегда экран подтверждения с кнопкой.

## Ключевые решения (зафиксированы)

- Отправка: **wa.me click-to-chat** — покупатель шлёт со своего номера, один тап.
- Заказы: **в БД всегда**, до открытия WhatsApp. Источник правды — БД, WhatsApp — канал.
- Хранение корзины: `localStorage` (ключ `tanar-cart`) + React Context.
- Позиция корзины = **SKU** (товар × цвет × размер), ключ — `skuId`. Цена/имя в
  корзине — снапшот для отображения; при оформлении сервер берёт цены из БД.
- Статусы заказа: `pending` (дефолт схемы, метка «Новый») → `confirmed` → `done` /
  `cancelled`. Дефолт схемы НЕ меняем.
- Человеческий номер заказа: колонка `orders.number` (serial, unique) — uuid в
  переписке не годится.
- QR: пакет `qrcode` (client-side `toDataURL`), блок `hidden md:block` — без UA-сниффинга.
- МП-ссылки живут ТОЛЬКО на карточке товара (в drawer корзины их нет).
- Цены МП на сайте не показываем — только кнопки-ссылки.
