## Learnings

### Итерация 1 (done 2026-06-05)
- **Канон происхождения названия зафиксирован:** text.md говорит «вдохновлено пиком Хан-Тенгри», docx раздел 5 — «образом Тенгри — Неба». Канон = **Хан-Тенгри** (совпадает с логотипом-горой). Трактовку «Тенгри/Небо» не используем нигде.
- **`layout.spec.ts:19` требует слово «Казахстан» в футере.** При замене нижней плашки «Алматы, Казахстан» обязательно сохранить «Казахстан» — иначе тест падает. Сейчас плашка: «ул. Розыбакиева, 205Д, Алматы, Казахстан».
- **e2e-грабли (телефон):** телефон есть и на `/contacts`, и в футере → `getByText('+7 701...')` ловит 2 элемента (strict mode violation). Скоупить: `getByRole('link', { name: '+7 701 744 38 73', exact: true })` для контактной секции.
- **e2e-грабли (FAQ `<details>`):** ответы внутри свёрнутого `<details>` имеют статус `hidden` → `toBeVisible()` падает. Либо кликнуть `<summary>` чтобы раскрыть, либо проверять `toBeAttached()` (в DOM).
- Константы `src/lib/site-contacts.ts` (объект `SITE_CONTACTS`) и `src/lib/faq.ts` (`FAQ_ITEMS`) — единый источник; футер уже их импортит. В итерации 2 они станут только сид-источником (см. шаг 7/8).
- `/contacts`, `/faq` собираются как **static** (`○`) — контент в коде, force-dynamic не нужен (станет dynamic в шаге 8 при чтении из БД).
- Оплата в FAQ — **нейтрально** («уточняйте при оформлении»), по решению пользователя (Kaspi явно не пишем до Фазы 3).
