# Step 3: References Audit (Patagonia + Arc'teryx)

> Статус: pending

## Цель

WebFetch-ом обойти 6 референсных URL и зафиксировать конкретные визуальные приёмы каждого бренда. Результат — материал для секции "Дизайн-аудит референсов" в финальном отчёте.

Фильтр качества: "это конкретный приём видимый на странице или общая отсебятина?" — если второе, не записывать.

## URL

**Patagonia:**
- https://www.patagonia.com/home/
- https://www.patagonia.com/shop/mens
- https://www.patagonia.com/stories

**Arc'teryx:**
- https://arcteryx.com/us/en
- https://arcteryx.com/us/en/shop/mens
- https://arcteryx.com/us/en/explore

## Что делать

1. WebFetch каждого URL с вопросом вида: *"Describe the visual language of this page: hero composition, color palette, lighting, presence/absence of human figures, subject scale, film grain / gloss, typography treatment."*

2. Для каждого бренда зафиксировать 5–8 конкретных визуальных приёмов:
   - Композиция hero (человек на переднем / дальнем плане, пейзаж без человека, крупный план)
   - Цветовая палитра (тёплая земляная / холодная каменная / смешанная)
   - Освещение (золотой час / синий час / overcast / harsh sun)
   - Присутствие человека (силуэт / лицо / action / нет)
   - Текстурность (film grain / матовость / глянец)
   - Контраст (low-key / high-key / matte)
   - Ракурсы и перспектива (уровень глаз / верхняя точка / эпический низ)

3. Для каждого приёма решить:
   - **Берём** для Tanar — если совпадает со стилистическим вектором (тёплый Patagonia + техничный Arc'teryx-штрих, centralasian вайб)
   - **Отвергаем** — если противоречит (яркий брендинг, спортивный экшн, альпийская швейцария)

4. Если WebFetch возвращает урезанный / JS-rendered контент — **явно отметить** "контент ограничен, опираюсь на <что увидел>" и работать с тем что есть. Не выдумывать приёмы.
