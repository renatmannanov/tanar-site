# Шаг 5.3: UX выбора источника + «сразу все»

> Зависит от: шаг 5.2 (генерация привязана к слоту)
> Статус: [ ] pending

## Задача

Довести UX заполнения пустого слота: когда источников **несколько**, показать кандидатов с миниатюрой (заказчик видит, из чего генерит), а не молча брать первый по приоритету. Плюс кнопка «сразу все» для пакетной генерации flat.

Решения зафиксированы: **звезда от базы** (источник = заполненный слот, целевой = где нажал; без галочки «база»); миниатюры источников показываем.

## 1. Кандидаты-источники для пустого слота

Для пустого слота собрать всех возможных кандидатов (из шага 5.2, таблица слот→рецепт) и показать списком/меню **с миниатюрой источника**:

```
Пустой flat_front[цвет X]:
   • [мин life_front X]   «сделать на белом» (рецепт 1, цвет точный)   ← приоритет 1
   • [мин flat_front Y]   «перекрасить из Y» (рецепт 2)                ← приоритет 2
   • [мин flat_front Z]   «перекрасить из Z» (рецепт 2)
   ── или ── [+ загрузить вручную]
```

- **Приоритет/сортировка кандидатов:** (1) рецепт 1 (свой life того же ракурса) — первым; (2) recolor из ближайшего по hex цвета. Расстояние hex считать простой функцией (client-safe, напр. сумма модулей разниц RGB-компонент). Это **подсказка-сортировка**, выбор за человеком.
- **Один кандидат** — всё равно показать его миниатюру (один клик, но видно из чего). Не генерить молча.
- **Ноль кандидатов** — только «+ загрузить вручную» (из 5.1).

### Раскрытие кандидатов

Инлайн в сетке станет шумно. Выбрать **раскрытие по клику** на пустой слот: клик → раскрывается компактный список кандидатов под/над слотом (или поповер). Один вариант, зафиксировать: **поповер/дропдаун от плитки слота** (не модалка на весь экран, не всегда-инлайн).

## 2. «Сразу все» (пакетная генерация flat)

Кнопка уровня варианта: «Сделать все на белом» — пройтись по заполненным `life_*` этого цвета, у которых пустой соответствующий `flat_*`, и сгенерировать flat по каждому (рецепт 1). 

- Источник каждого — свой `life_<view>` (ракурс наследуется).
- Идёт последовательно (не параллелить — стоимость/рейт-лимит Gemini); показывать прогресс («2 из 3…»).
- **Превью каждого результата перед сохранением — это шаг 6.** В шаге 5.3 «сразу все» может складывать в слоты сразу (как одиночная генерация сейчас); апрув накрутится в шаге 6 единообразно для одиночной и пакетной.
- Кнопка видна только если есть ≥1 заполненный `life_*` с пустым парным `flat_*`.

## 3. Что НЕ делаем в этом шаге

- Превью/апрув результата — шаг 6.
- Рекомендацию «в каком порядке красить цвета» / «какой цвет базовый» — НЕ делаем (польза под вопросом; recolor по hex работает одинаково для далёких цветов — подтверждено тестом красный→navy). Если понадобится — отдельный шаг после.

## Тесты

- **e2e (мок Gemini):**
  - пустой `flat_front` при 2 цветах с источниками → клик по слоту раскрывает список ≥2 кандидатов с миниатюрами → выбор конкретного → генерация в слот.
  - «Сделать все на белом»: 2 заполненных `life_*` + пустые `flat_*` → клик → 2 flat появляются в слотах.
- `npm run build/typecheck/lint/test:e2e` зелёные.
