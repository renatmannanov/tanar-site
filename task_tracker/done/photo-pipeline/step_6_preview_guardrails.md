# Шаг 6: Превью + апрув + метка ИИ + лимит (под слоты)

> Зависит от: шаги 5.1–5.3 (сетка слотов, генерация из слота, выбор источника)
> Статус: [ ] pending

## Задача

Накрутить на slot-генерацию защиту от ошибок заказчика: ничего не публикуется без апрува; перезапись занятого слота — с подтверждением; метка «ИИ» в админке и на витрине; лимит генераций (выключен). Переписано под слоты (НЕ плоский список).

## 1. Превью + апрув (главная защита)

Результат генерации **НЕ пишется в `media_assets` автоматически**. Поток для одиночной и пакетной генерации:

```
[слот: заполнить] → генерация → ПРЕВЬЮ результата → [ Оставить ] [ Перегенерировать ] [ Отмена ]
```

- **Технически (решено 2026-06-05 — base64 клиенту):** разделить экшены на две фазы.
  - `generate*` → НЕ грузит на диск/в БД. Возвращает `{ previewDataUrl: string }` — сгенерированный webp как data-URL (`data:image/webp;base64,…`). Размер webp 1600px ≈ 100–300 КБ, в base64 ≈ +33% — терпимо для одиночной и для пакетной на 6 слотов.
  - `approveGeneratedAction(dataUrl, target)` → декодит base64 → `mediaStore.upload` (sharp+webp+строка) с `aiGenerated:true`. Зовётся ТОЛЬКО на «Оставить». Нет осиротевших webp на диске (на диск пишем лишь при апруве).
  - Превью держит КЛИЕНТ (в state) — никакого временного файла/таблицы на сервере. «Отмена» = клиент выбросил data-URL, на сервере ничего не осталось.
- «Перегенерировать» → новый вызов Gemini, старое превью отбрасывается.
- «Отмена» → ничего не сохранено.
- **e2e-ловушка:** превью-картинка НЕ должна попадать в `ul li img` галереи/сетки до апрува — иначе `admin-crud-media.spec.ts` посчитает её сохранённой. Превью рендерить ВНЕ `<ul>` занятых слотов (отдельный блок/поповер).

## 2. Перезапись занятого слота (двойное подтверждение)

Заказчик решил перегенерировать слот, где уже есть фото:

```
[занятый слот → перегенерировать] → генерация → ПРЕВЬЮ → [Оставить] →
   → подтверждение «Заменить текущее фото в слоте <…>?» → [Заменить] / [Отмена]
```

- Сначала **апрув результата** (нравится ли новое), потом **апрув замены** (точно заменить старое). Старое фото не трётся, пока оба не подтверждены.
- На «Заменить»: `mediaStore.upload` нового + `mediaStore.remove` старого (или upload с тем же слотом и удаление прежнего) — в логичном порядке, без гонки.

## 3. Метка «сгенерировано ИИ»

- **Поле в БД:** добавить `ai_generated boolean default false` в `media_assets` через миграцию drizzle (`npm run db:generate` после правки `schema.ts`, затем `npm run db:migrate`). Минимальное изменение схемы. **Старые строки не засыпаем задним числом** (решено 2026-06-05): на dev данные пересоздаются `db:seed`, прод-фото ещё не залиты → default `false` достаточно. Признак ставится только на НОВЫХ генерациях.
- `mediaStore.upload` принимает `aiGenerated?: boolean` (тип `MediaUploadInput` уже расширяем — в шаге 5 поле было удалено как преждевременное, вернуть). `approveGeneratedAction` ставит `true`, ручная загрузка — `false` (default).
- **Чистка костыля `model:'flat'` как признака ИИ:** сейчас flat-генерация и `setRole` пишут `model:'flat'` (служит и типом, и косвенным «это сделал движок»). С появлением честного `ai_generated` — `model:'flat'` остаётся ТОЛЬКО как тип/служебная метка движка, признак «ИИ» читаем из `ai_generated`. Не смешивать в показе бейджа.
- `MediaAsset` (client-тип) получает `aiGenerated?: boolean`; `mapAssetRow` мапит колонку.
- **Показ:** бейдж «ИИ» на фото в админке (на миниатюре слота) + ненавязчивый значок «изображение создано ИИ» на витрине (согласовано — показываем покупателю).

## 4. Лимит фото и лимит генераций

- **SOFT_MAX=8** уже есть; 6 слотов < 8 — не мешает. Оставить как страховку.
- **Лимит генераций:** счётчик вызовов Gemini (константа/env, напр. `PHOTOGEN_DAILY_LIMIT`), **по умолчанию выключен** (∞ / флаг off). Код готов включить одной строкой; прокомментировать где. На тесте не мешает.

## Тесты

- **e2e (мок Gemini, `PHOTOGEN_FAKE=1`):**
  - генерация в пустой слот → превью → «Оставить» сохраняет (в сетке появляется фото); «Отмена» не сохраняет (счётчик `ul li` не растёт).
  - занятый слот → перегенерация → превью → «Оставить» → подтверждение замены → старое заменено (по url/id).
  - сгенерированное фото имеет признак ИИ (бейдж в админке; значок на витрине).
- `npm run build/typecheck/lint/test:e2e` зелёные; миграция применяется на чистой БД.
