# Шаг 3: Caddyfile (reverse-proxy + авто-SSL)

> Зависит от: нет (но `up` целиком проверяется вместе с шагом 2).
> Статус: [ ] pending

## Задача

Caddy терминирует TLS и проксирует на `web:3000`. Авто-SSL Let's Encrypt по домену `.kz`.

### `Caddyfile` (новый, в корне)
Прод-конфиг:
```
{$DOMAIN} {
    reverse_proxy web:3000
    encode gzip zstd
}
```
- `{$DOMAIN}` — подставляется из env (домен задаётся в `.env`, напр. `DOMAIN=tanar.kz`). Caddy сам выпустит и обновит сертификат для этого домена.
- http→https редирект Caddy делает АВТОМАТИЧЕСКИ при наличии домена (не localhost) — отдельная директива не нужна.
- **Тело запроса:** НЕ ставить `request_body max_size` (по умолчанию Caddy не режет → 10mb-загрузки фото проходят). Если позже понадобится лимит — ставить ≥ `10MB`.

### Отладка на проде без сжигания LE-лимита (ФИКС ревью)
Let's Encrypt лимитирует неудачные выпуски (5 failed/час/домен) и успешные (≤5 дублей сертификата/неделю). Если на шаге 7 придётся несколько раз перезапускать Caddy (отладка DNS/портов) — каждый раз дёргается ACME. Чтобы не словить блокировку домена на час, при отладке временно использовать **staging-issuer** LE (нелимитированный, но сертификат невалидный — только для проверки, что выпуск проходит):
```
{$DOMAIN} {
    reverse_proxy web:3000
    tls {
        ca https://acme-staging-v02.api.letsencrypt.org/directory
    }
}
```
После успешной проверки — убрать блок `tls`, перезапустить caddy (получит уже боевой сертификат). Зафиксировано: staging — ТОЛЬКО на время отладки шага 7, в коммит идёт прод-вариант без `tls`-блока.

### Локальный режим (для теста шага 2 без реального домена)
Для локальной проверки стека (нет публичного домена/DNS) — временный Caddyfile-вариант:
```
:80 {
    reverse_proxy web:3000
}
```
Это позволяет проверить проксирование на `http://localhost` без TLS. Зафиксировано: держать прод-Caddyfile как основной; для локального теста временно подменять на `:80`-вариант (не коммитить подмену). На проде — `{$DOMAIN}`.

## Тесты
- `caddy validate` на конфиге (синтаксис).
- Локально (`:80`-режим): `http://localhost` отдаёт витрину через caddy→web.
- На проде (шаг 7): https с валидным сертификатом.

## Команды для верификации
```powershell
# Валидация синтаксиса Caddyfile (через образ caddy):
docker run --rm -v ${PWD}/Caddyfile:/etc/caddy/Caddyfile caddy:2-alpine caddy validate --config /etc/caddy/Caddyfile --adapter caddyfile
# Локальная проверка проксирования — в связке со стеком шага 2 (:80-режим).
```
