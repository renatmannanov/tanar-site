> Статус: done

> Тип: фича (завершение создания товара + полнота витрины)

## Цель

Довести создание товара и витрину до полноты: автоген slug при создании, видимость витрины по статусам (draft/archived скрыты от покупателя), показ покупателю существующих данных (размеры, уход, бейдж), и редактор характеристик (specs) в админ-форме. После этого заказчица может полностью завести товар из админки, а покупатель видит всю релевантную инфу.

## Архитектурные решения (зафиксированы с пользователем, без развилок)

- **Slug:** автоген из `name` транслитерацией кириллицы. Поле видимое, **read-only** (юзер не правит руками). Только на create; edit — slug read-only и неизменен (как сейчас). **Дубль slug → автоинкремент суффикса** (`kurtka` занят → `kurtka-2`, `-3`...): сервер подбирает уникальный (`ensureUniqueSlug`), заказчица о slug не думает. Пустой slug → кнопка «Создать» disabled.
- **Статусы витрины:** `published` + `coming_soon` — видны покупателю; `draft` + `archived` — скрыты (нет в каталоге/related/featured; прямой заход → 404). Админка видит ВСЕ статусы. **Решение (без развилок):** ОТДЕЛЬНЫЕ storefront-функции чтения (`getStorefront*`), фильтрующие по статусу; старые `getAllProducts`/`getProductBySlug`/`getProductsByCategory`/`getRelatedProducts` остаются как есть (их зовёт админка). Витринные страницы переключаются на storefront-варианты. Новый товар = draft (уже так в EMPTY_INPUT).
- **Размеры на витрине:** список размеров активного цвета (`sku.size` + `sku.ruSize` если есть, формат «M / 46»). БЕЗ наличия (stockQty — Фаза 2).
- **care:** блок «Уход» на странице товара, если заполнено.
- **label.badge:** на странице товара (рядом с названием/ценой) И на карточке каталога (ProductCard, уголок).
- **specs-редактор:** массив пар {label, value} в ProductForm (добавить/удалить/править строки). specs уже текут через маппер+схему — нужен только UI. specs НЕ дублируют label (label = значок {badge,sub}; specs = таблица характеристик).

## НЕ входит (бэклог)
