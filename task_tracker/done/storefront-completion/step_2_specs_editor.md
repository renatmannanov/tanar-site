# Шаг 2: Редактор характеристик (specs) в ProductForm

> Зависит от: **шаг 1** (та же `ProductForm.tsx` — ПОСЛЕ шага 1, не параллельно).
> Статус: [ ] pending

## Задача

Добавить в ProductForm редактор массива `specs` (характеристики товара) — пары `{label, value}`. Сейчас specs показываются на витрине (`<dl>`), НО у всех боевых товаров пустые, и в админке нет поля их редактировать.

> **specs уже текут через данные:** `productInputSchema.specs` (zod) есть, `productToInput` пробрасывает `specs: p.specs`, `mapProduct` читает `row.specs`. Правки маппера/схемы НЕ нужны — только UI в форме.

### ProductForm (`src/components/admin/ProductForm.tsx`)
- patch-функции для specs (по аналогии с вариантами):
  - `addSpec()` — добавить `{ label: '', value: '' }` в `form.specs`.
  - `removeSpec(i)` — удалить i-ю.
  - `patchSpec(i, p)` — обновить label/value i-й.
  - Учесть: `form.specs` может быть `undefined` (тип `ProductInput['specs']` optional) — работать через `(form.specs ?? [])`.
- UI-секция «Характеристики» (между «Уход» и «Цвета и размеры», или после блока label — на усмотрение вёрстки, но ОТДЕЛЬНОЙ секцией):
  - Заголовок + подсказка «Материал, вес, мембрана и т.п. — показываются на странице товара таблицей».
  - Для каждой пары: два инпута (label «Характеристика», value «Значение») + кнопка «×» удалить строку (обычная, без ConfirmButton — правка только в форме, как у размеров).
  - Кнопка «+ Характеристика» — добавляет пустую строку.
  - Пустой список — допустим (specs optional).
- **Фильтрация пустых пар при submit (зафиксировано):** в onSubmit формировать payload, отфильтровав пары, где `label.trim()===''` ИЛИ `value.trim()===''` (обе должны быть непустыми). Так пустые/полупустые строки-характеристики не попадут в БД. Применить фильтр к `form.specs` при сборке payload (не мутируя стейт).
