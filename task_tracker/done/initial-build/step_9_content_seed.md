# Шаг 9: Seed контента (продукты + посты)

> Зависит от: шаги 5, 7, 8 (типы и утилиты уже есть)
> Статус: [ ] pending

## Задача

Наполнить `src/data/products.ts` продуктами и `content/blog/` постами. **Это единственный шаг который генерирует контент. Всё остальное — каркас.**

### Порядок действий

1. **Продукты `src/data/products.ts`** — полностью перезаписать, минимум 20 штук:
   - По 5–7 продуктов в каждой из 4 категорий: `jackets`, `backpacks`, `accessories`, `t-shirts`
   - **Первый продукт в массиве ОБЯЗАТЕЛЬНО должен иметь slug `shell-jacket-khan`** (категория `jackets`) — на этот slug завязан smoke-тест в step_11.
   - Примеры названий (смешать, добавить свои в том же духе):
     - jackets: "Куртка Хан Шелл" (slug `shell-jacket-khan`, обязательный), "Парка Тенгри", "Анорак Алатау", "Ветровка Турген", "Жилет Иле"
     - backpacks: "Рюкзак Чарын 40L", "Штурмовой Мерке 28L", "Городской Шымбулак 18L", "Треккинговый Кольсай 55L"
     - accessories: "Шапка Бурабай", "Перчатки Заилийский", "Бафф Каскелен", "Ремень Капшагай"
     - t-shirts: "Футболка Логотип", "Футболка Хан Тенгри", "Лонгслив Восход", "Футболка Горный контур"
   - Цены: реалистичные в KZT (25 000 – 180 000)
   - slug: транслит латиницей через дефис (`shell-jacket-khan`, `backpack-charyn-40l`)
   - gradient: взять из `OUTDOOR_GRADIENTS`, детерминированно от slug (можно через `gradientFromString`) или вручную, но из одобренного списка
   - description: 2 предложения на продукт, осмысленные, на русском
   - specs: 3–5 пар на продукт, примеры:
     - "Материал" → "Gore-Tex Pro 3L"
     - "Вес" → "460 г"
     - "Водостойкость" → "28 000 мм"
     - "Объём" → "40 л"
     - "Посадка" → "Регулярная"
   - **Типизировать через `satisfies Product[]`** чтобы tsc проверил каждый элемент

2. **Посты блога** — создать 6 файлов в `content/blog/`. Удалить `_placeholder.mdx`.
   - Имена файлов совпадают со slug: `content/blog/<slug>.mdx`
   - **Первый пост (по сортировке — самая свежая дата) ОБЯЗАТЕЛЬНО должен иметь slug `khan-tengri-ascent`** — на этот slug завязан smoke-тест в step_11.
   - Темы (6 штук):
     1. "Восхождение на Хан Тенгри: что взять" (slug `khan-tengri-ascent`, самая свежая дата) — гайд
     2. "Как выбрать куртку для Тянь-Шаня" — гайд
     3. "Треки Казахстана которые стоит пройти" — подборка
     4. "История бренда Tanar" — сторителлинг
     5. "Тестируем рюкзак в Кольсайских озёрах" — обзор
     6. "Философия эко-производства Tanar" — манифест
   - Каждый пост: frontmatter (все 6 полей) + 400–800 слов на русском в теле. **Не латинский lorem!** Связный осмысленный текст, можно простой и повторяющийся, но на тему поста. Нормально использовать повторения, списки, цитаты, заголовки H2/H3.
   - Даты постов: разные, последние 3–4 месяца (от 2026-01 до 2026-04). `khan-tengri-ascent` — самая свежая (например, `2026-04-18`).
   - gradient: разные, из одобренного списка
   - author: "Команда Tanar" или конкретные имена вроде "Алия Нурлан", "Данияр Касымов"
   - **Никаких тегов `<img>`** внутри MDX (или только с alt — Placeholder всё равно заменит)
   - **Удалить placeholder** идемпотентно (с `-f` — не упадёт если файла уже нет):
     ```bash
     rm -f content/blog/_placeholder.mdx
     ```

3. **Проверить что ранние fallback-и больше не срабатывают**:
   - `/catalog` показывает реальные продукты, сообщение "Скоро здесь появятся товары" не видно
   - Home → `FeaturedProducts` показывает 4 реальных продукта
   - Home → `LatestPosts` показывает 3 реальных поста
   - `/blog` показывает 6 постов

4. Коммит:
   ```bash
   git add -A
   git commit -m "feat(content): seed 20+ products and 6 blog posts"
   ```

## Команды для верификации

```bash
npm run typecheck
npm run lint
npm run build
npm run test:e2e
```

Файлы и контент:

```bash
# Минимум 20 продуктов (считаем slug-строки)
test $(grep -c "slug:" src/data/products.ts) -ge 20

# Обязательный slug для smoke-теста присутствует
grep -q "slug: *['\"]shell-jacket-khan['\"]" src/data/products.ts

# 6 MDX постов в content/blog/, без underscore (find работает кросс-платформенно)
test $(find content/blog -maxdepth 1 -name "*.mdx" ! -name "_*" | wc -l) -eq 6

# Обязательный пост для smoke-теста
test -f content/blog/khan-tengri-ascent.mdx

# _placeholder.mdx удалён
test ! -f content/blog/_placeholder.mdx
```

Браузер-проверка через Playwright (будет в step_11):
- `/catalog` содержит ≥20 `data-testid="product-card"`
- `/blog` содержит ровно 6 `data-testid="blog-card"`
