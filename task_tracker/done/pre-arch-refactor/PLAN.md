# Pre-architecture refactor

> Статус: done

> Тип: рефакторинг

## Цель

«Причёсывающий» рефактор текущего кода **до** миграции на новую модульную архитектуру (БД, Фаза 0). Убираем структурную грязь, которая иначе поедет в БД-схему: дублирование категорий, представление (Tailwind-градиенты) в данных, файловую конвенцию путей к картинкам, прямые импорты данных в обход слоя. Это создаёт чистую «точку подмены» для будущего перехода на БД.

## Рамки (что НЕ делаем)

- НЕ миграция на БД, НЕ async-функции, НЕ добавление размеров/SKU, НЕ новая модель Product. Всё это — Фаза 0, отдельно.
- Только дешёвый структурный рефактор на текущей синхронной кодовой базе.
- Внешнее поведение сайта не меняется. Все существующие e2e остаются зелёными.
