## Задача

Обновить Playwright-тесты под боевой каталог. Демо-slug'и и счётчики больше не валидны.

### `e2e/catalog.spec.ts`
- Тест `all filter chips are present`: текущий массив в файле — `['Все', 'Куртки', 'Худи', 'Футболки', 'Штаны', 'Шорты']`. Заменить ЦЕЛИКОМ на `['Все', 'Куртки', 'Брюки', 'Шорты', 'Футболки', 'Поло']`. (Внимание: «Куртки» и «Шорты» остаются — не потерять их; уходят «Худи», «Штаны»→«Брюки», добавляется «Поло».)
- Тест `clicking "Куртки" chip filters by category` — остаётся (jackets есть, ярлык «Куртки» есть). Проверить что `category=jackets` в URL и активный чип. Без изменений по сути.

### `e2e/smoke.spec.ts`
- `PRODUCT_SLUG = 'shell-jacket-khan'` → `'jacket-sv7-goretex'` (боевой slug из снапшота).
- Тест `coming-soon products show "Скоро" badge` на `/catalog/pants-charyn`: в боевом каталоге **coming_soon товаров НЕТ**. Решение: **удалить этот тест** (нет данных для него; вернётся, если заказчица заведёт coming_soon через админку — но это не часть Плана A). Зафиксировать удаление в коммите.
- `catalog shows all 10 product cards`: `10` → `12`.
- `catalog filter by jackets`: остаётся; проверяет `data-category='jackets'` на всех карточках. Боевых курток 4 → `count` будет 4, тест проверяет `>0` и атрибут — менять не нужно (но можно уточнить до `toBe(4)` для строгости — **оставить `>0`**, чтобы тест не ломался при будущем изменении каталога).
- `metadata titles` — `/catalog/${PRODUCT_SLUG}` теперь боевой slug. Заголовок matches `/Tanar$/` — остаётся валидным.
- URL list в `smoke` describe: `/catalog/${PRODUCT_SLUG}` подхватит новый slug автоматически.

> Принцип: счётчики, привязанные к точному числу боевых товаров (12), фиксируем числом там, где это смысловая проверка («весь каталог виден»). Фильтры по категории — оставляем `>0` (устойчивее к правкам каталога).
