# Шаг 5.1: Модель 6 слотов + сетка 2×3

> Зависит от: шаг 5 (движок photogen + server actions + черновой UI — выполнен)
> Статус: [ ] pending

## Задача

Заменить плоский список фото в `VariantPhotos` на **структурированную сетку 6 слотов** на каждый цвет (variant). Слоты гарантируют `role`+`view` у каждого фото — это фундамент, который лечит косяк с `view=null` (см. шаг 5.2). Схему БД **не менять** (`role`/`view` уже есть в `media_assets`).

### 6 слотов (матрица 2×3)

```
              front        side         back
  ┌──────────────────────────────────────────────┐
  живое   │ life_front  │ life_side  │ life_back  │   role='lifestyle'
  на белом│ flat_front  │ flat_side  │ flat_back  │   role='flat'
  └──────────────────────────────────────────────┘
```

Слот = пара `(role, view)`. «Слот занят» = есть строка `media_assets` с этим `variantId` + `role` + `view`. В этом шаге слот хранит **максимум одно фото** (перезапись/несколько — шаг 6).

### Что делаем

1. **Хелпер слотов (client-safe).** В `@/core/media/client` добавить:
   - тип `PhotoSlotKey = 'life_front' | 'life_side' | 'life_back' | 'flat_front' | 'flat_side' | 'flat_back'`;
   - константу `PHOTO_SLOTS` (упорядоченный список 6 слотов с `role`/`view`/русской меткой);
   - функцию `slotOf(asset): PhotoSlotKey | null` — определяет слот фото по `role`+`view` (нет `view` → null = «вне сетки»);
   - функцию `assetsBySlot(assets): Record<PhotoSlotKey, MediaAsset[]>` — группировка.
   - НЕ тянуть sharp/db — только типы и чистые функции (как `urlForWidth`/`srcSetFromUrl` уже там).

2. **Загрузка в конкретный слот.** `onPick` в `VariantPhotos` сейчас шлёт FormData без `role`/`view`. Добавить: при клике на пустой слот запоминаем целевой `(role, view)` → `onPick` кладёт их в FormData → `uploadVariantImageAction` пробрасывает в `mediaStore.upload({ role, view })`. Бэкенд (`store.ts`, `MediaUploadInput`) уже принимает `role`/`view` (шаг 5).
   - `flat`-слот руками тоже можно (вдруг есть готовый студийный) → загрузка с `role:'flat'`.
   - `life`-слот → `role:'lifestyle'`.

3. **Сетка 2×3 вместо `<ul>`-списка.** Отрендерить `PHOTO_SLOTS` сеткой: занятый слот = миниатюра + крестик удаления; пустой слот = плитка «+ загрузить» с подписью ракурса/типа.
   - Кнопки генерации (шаг 5.2) пока оставить как есть (черновые) — этот шаг только про сетку и загрузку. Связать с слотами — шаг 5.2.
   - **e2e-ловушки** (`admin-crud-media.spec.ts`): file input берётся `.first()`; превью считаются `ul li` / `ul li img`; кнопка удаления — name `×`. Сетку слотов МОЖНО рендерить как `<ul><li>` (тогда существующий e2e продолжит считать `ul li`), но число занятых ячеек должно совпадать с числом загруженных фото. Проще: сохранить `<ul class="grid"><li>` для ЗАНЯТЫХ слотов (как сейчас), пустые слоты-плитки — отдельным блоком вне этого `<ul>`. Тогда `ul li`-счётчики в e2e не поедут.

4. **Витрина: порядок life→flat.** `listProductImages` (`@/core/media/index.ts`) сортирует по `(variantId, sortOrder)`. Изменить сортировку для витрины так, чтобы внутри варианта сначала шли `role='lifestyle'`, потом `role='flat'`, внутри роли — по `view` (front→side→back), затем sortOrder. Проверить обе функции (`listProductImages`, `listProductImagesForProducts`) — обе питают витрину.
   - Админка (`edit/page.tsx`) использует ту же `listProductImages`, но группирует по слотам сама — порядок ей неважен.

5. **Убрать ручную сортировку (стрелки ←/→).** Решено (2026-06-05): порядок на витрине полностью определяется слотами (role→view), ручной reorder больше не нужен и конфликтует со слот-порядком. В `VariantPhotos` удалить функцию `move` + кнопки ←/→ (`Левее`/`Правее`). `reorderVariantImagesAction` (media-actions) и `mediaStore.reorder` (store) **оставить в коде мёртвыми** (безвредны, не зовутся из UI) — не выпиливать, чтобы не трогать контракт store и не плодить лишний диф.

6. **Legacy-фото вне сетки (дешёвая страховка).** Решено: данные с `view=null` (старая ручная загрузка) **не мигрируем** — на dev они пересоздаются `db:seed`, прод-фото ещё не залиты. НО чтобы фото с `slotOf===null` не «пропало» из админки (сетка рендерит только 6 слотов, а на витрине оно ещё видно), добавить тонкий блок «вне сетки» — показывать **только если такие ассеты есть** (для чистой БД блока нет). В блоке: миниатюра + крестик удаления + (опц.) кнопка «назначить слот» позже. Минимум — миниатюра+удаление, чтобы owner мог разрулить руками.
