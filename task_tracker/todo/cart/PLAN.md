# Корзина (cart)

> Статус: pending
> Дата: 2026-05-23
> Тип: фича

## Цель

Добавить на сайт корзину: посетитель набирает товары (с выбранным цветом), открывает корзину в drawer, управляет позициями (удалить, изменить количество, видеть стоимость позиции и общую сумму) и отправляет заказ в мессенджер заказчицы. Состояние корзины живёт в `localStorage` + React Context (без бэкенда и БД, по CLAUDE.md). Отправка заказа идёт через серверный Next.js route handler с абстракцией канала уведомления (notifier): на старте — заглушка-лог + Telegram как первая реализация.

## Архитектурные решения (зафиксированы, без развилок)

- **Хранение:** `localStorage` ключ `tanar-cart`, синхронизация через React Context (`CartProvider` в `layout.tsx`).
- **Позиция корзины (line item):** уникальна по паре `slug + colorId`. Повторное добавление той же пары увеличивает `qty`. Товары без варианта (нет `variants`) — ключ `slug + ''`.
- **comingSoon товары** (нет цены, `comingSoon: true`) — нельзя класть в корзину. Кнопка "В корзину" на них не показывается.
- **UI открытия корзины:** drawer (выезжает справа), не отдельная страница. Иконка-кнопка со счётчиком в Header открывает drawer.
- **Отправка заказа:** клиент шлёт POST на `/api/order` с телом заказа → route handler валидирует → вызывает `notify(order)` → notifier по `env`-переменной выбирает реализацию. На старте default = `console` (лог в серверную консоль), при наличии `TELEGRAM_BOT_TOKEN` + `TELEGRAM_CHAT_ID` = Telegram.
- **Контакт покупателя:** в форме отправки заказа одно обязательное поле — имя + телефон/телеграм (одна строка `contact`, обязательная, min 5 символов). Без email, без адреса (детали уточняет заказчица в мессенджере).
- **Валюта:** только KZT, формат через существующий `formatPrice` из `src/lib/product.ts`.
