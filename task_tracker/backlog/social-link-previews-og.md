# Социальные превью ссылок (Open Graph / Twitter Cards)

> Тип: фича / SEO + соцшеринг
> Статус: backlog
> Заявлено: 2026-06-10

## Контекст

При вставке ссылки на tanar.kz в Telegram/WhatsApp/соцсети рисуется карточка-превью.
Сейчас (на момент заявки) подтягиваются только `<title>` и `<description>` из дефолтных
Next-метаданных — заголовок «Tanar — Outdoor-бренд из Казахстана» и описание видны,
**но нет картинки** (`og:image`), поэтому карточка «голая».

Задача: сделать богатые превью — с обложкой, корректным заголовком/описанием на каждый
тип страницы, и `twitter:card` для X.

## Что сделать

### 1. Глобальные OG-метаданные (`src/app/(public)/layout.tsx` или корневой `layout`)
В Next 15 App Router — через экспорт `metadata`:
- `openGraph`: `title`, `description`, `siteName: 'Tanar'`, `url`, `locale: 'ru_KZ'`, `type: 'website'`, `images: [{ url, width: 1200, height: 630, alt }]`
- `twitter`: `card: 'summary_large_image'`, `title`, `description`, `images`
- `metadataBase: new URL('https://tanar.kz')` — чтобы относительные пути картинок резолвились в абсолютные (обязательно для OG).

### 2. OG-картинка (обложка по умолчанию)
- Вариант A (просто): статичный баннер `public/og/default.png` 1200×630 — лого TANAR + фон в фирменной палитре.
- Вариант B (динамика): `next/og` `ImageResponse` — рендерить баннер на лету (`app/.../opengraph-image.tsx`). Пригодится для товаров/постов.

### 3. Динамические метаданные на товарах и блоге
- `catalog/[slug]/page.tsx`: `generateMetadata` → название товара, цена/категория в описании,
  **первое фото варианта** как `og:image` (через `@/core/media` `listProductImages`). Фолбэк —
  дефолтная обложка, если фото нет (учесть градиент-фолбэк витрины).
- `blog/[slug]/page.tsx`: `generateMetadata` → title/excerpt поста из frontmatter, обложка поста
  (сейчас посты на градиентах — можно сгенерировать og-image из заголовка через `next/og`).

### 4. Проверка
- Билд проходит (`npm run build`), typecheck чистый.
- Прогнать ссылку через debugger-ы: Telegram (просто вставить), [opengraph.xyz](https://www.opengraph.xyz/),
  Facebook Sharing Debugger, Twitter Card Validator.
- Проверить абсолютность URL картинок (частая ошибка — относительный путь → соцсеть не грузит).

## Подводные камни

- **`metadataBase`** обязателен, иначе `og:image` уйдёт относительным и не отрисуется.
- Соцсети **кешируют** превью — после деплоя дергать debugger, чтобы сбросить кеш (особенно Facebook/Telegram).
- Картинки товаров отдаются на проде **через Caddy с volume** (см. CLAUDE.md, прод-деплой) — проверить,
  что og:image-URL доступен публично без авторизации и отдаёт правильный content-type.
- На force-dynamic страницах `generateMetadata` тоже выполняется на каждый запрос — читает БД,
  обернуть в try/catch как и сам рендер (build без DATABASE_URL не должен падать).
