## Контекст

В рамках photos-integration plan проверили гипотезу: можно из lifestyle-кадра (человек в куртке на фоне гор) получить **студийный flat-shot на белом фоне** через nano-banana. Гипотеза подтверждена 100% — front, side, back все три ракурса работают, цвета можно перекрашивать через hex с сохранением геометрии.

Демо для заказчика: `/catalog/shell-jacket-khan?color=red` — **галерея сначала показывает 3 flat-shot, потом 3 lifestyle-shot**. Это пилот для **Куртки Хан Шелл, красный цвет**.

## Что в backlog

### 1. Скилл для быстрой генерации flat-shots

Сейчас процесс ручной: я вызываю `mcp__nano-banana__edit_image` индивидуально для каждой картинки, потом вручную через `node -e "import('sharp')..."` гоняю sharp, потом руками раскладываю по папкам.

**Цель:** сделать `/skill flat-shot <slug> [color]` (или вариант через npm-скрипт), которая:
- Читает `products.ts`, находит товар
- Если у `variant` уже есть `hasFlatShots: true` — пропускает или переделывает (флаг `--force`)
- Берёт исходник из `assets/products/<slug>/<color>/<model>/1_{front,side,back}.jpg`
- Прогоняет через nano-banana с правильным промптом (рецепт 1.a/1.c из [nano-banana-recipes.md](../../internal/docs/nano-banana-recipes.md))
- Прогоняет результат через sharp в 3 размерах
- Кладёт в `public/images/products/<slug>/<color>/<view>-flat-*.webp`
- Помечает `hasFlatShots: true` в products.ts (через AST или ручную инструкцию)

**Альтернатива #1**: если **уже есть** flat-shot одного цвета (baseline), и нужно **только перекрасить** в другие — вызывает рецепт 2 на готовом baseline + hex из `products.ts`. Это намного дешевле и даёт **одинаковую геометрию** между цветами одной модели.

**Альтернатива #2**: расширить как полноценный pipeline-скилл, не отдельный — вместе с задачей [assets-pipeline-extension](./assets-pipeline-extension.md).

### 2. Прогнать остальные товары и цвета

После скилла — массово запустить на:

- **Куртка Хан Шелл** — осталось 3 цвета (darkblue, white, yellow). Red уже сделан как пилот.
- **Парка Тенгри** — 2 цвета × 2 модели (orange/girl, orange/man, white/man) = 3 lifestyle-источника
- **Худи Алатау** — 3 цвета × разные модели = 4 lifestyle-источника
- **Худи Турген** — 2 цвета (lightpink, red), оба только girl
- **Футболка Танар** — 2 цвета (blue, pink), оба только girl

Итого ~14 lifestyle-источников × 3 ракурса = **~42 flat-shot**.

**Стратегия:** для каждой модели сделать **один baseline** (тот цвет, что уже на лучшей фотке у фотографа), потом **перекрасить** в остальные цвета через рецепт 2. Минимум API-вызовов и **максимальная визуальная единообразность** между цветами.

**Узкие места:**
- Трикотаж (худи, футболка) не тестирован, поведение nano-banana может отличаться
- Принты на футболках могут глючить при перекраске
- Подкладки/манжеты контрастного цвета — нужен отдельный подход

После этого — переключить листинг каталога: показывать **flat-shot вместо lifestyle** для главной фотки в карточке (Patagonia-стиль). Lifestyle оставить для галереи на странице товара. Это даст **резкое улучшение визуальной целостности каталога**.

### 3. Подумать как продавать эту фичу как отдельный продукт

То что мы делаем здесь — **востребованная штука**:
- Фотосессия товаров на белом фоне в студии: 5-15$ за SKU + день фотографа + время
- Через nano-banana: ~$0.04 за вызов × 3 ракурса = **$0.12 за SKU**, минуты на товар. Качество по нашему тесту — каталожное.

**Кому продавать:**
- Маленькие казахстанские бренды одежды/обуви (как Tanar) с lifestyle-фотками, но без бюджета на студию
- Дропшипперы и реселлеры на маркетплейсах (Wildberries, Kaspi, Ozon), у которых требования к flat-shot но фоток нет
- Стартапы, которые делают первый каталог и хотят выглядеть как Arc'teryx без затрат

**Что нужно для продукта:**
- UI: загрузка lifestyle-кадра → выбор цветов (hex) → результат
- Pipeline скрытый, цены сверху × N (например, $1 за SKU вместо $0.12, маржа на промпт-инжиниринг и UX)
- Автокроп / автоматический поиск товара на кадре (упрощение для пользователя)
- Готовые "пресеты" под маркетплейсы (WB требует X×Y, Ozon другое)

**Гипотезы:**
- Можно собрать MVP за неделю: Next.js + nano-banana API + Stripe
- Канал — Instagram-таргет на казахстанские/российские бренды одежды
- Цена $1-3 за товар, хороший аналог — `removebg.com`, но мы делаем **больше** (форма + цвет + ракурс)

**Это отдельный продукт**, не функция Tanar-сайта. Запускать после того как обкатаем пайплайн на собственном каталоге (пункт 2).
