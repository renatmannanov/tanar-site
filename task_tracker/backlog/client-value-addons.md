# Client value add-ons (доп.услуги для заказчицы)

> Тип: набор идей для апсейла + AI-обвязка
> Статус: backlog (идеи, не планы)
> Когда брать: после сдачи v1 сайта, по одной — каждая идея = отдельный апсейл

## Контекст

Заказчица — небольшой outdoor-бренд (Tanar), бюджет ограничен. Базовый сайт делаем как фундамент, но рядом есть несколько штук, которые:
- занимают часы (не дни) на нашей стороне,
- стоят у фотографа/SMM/разработчика десятки-сотни тысяч тенге,
- лучше всего раскрываются если **сразу заворачивать AI-обвязку**, чтобы это была не разовая услуга, а воспроизводимый инструмент.

Ниже 4 направления + мысли как каждое "обИИшить".

---

## 1. Оптимизация фото для веба

**Базовая ценность:** конвертация в WebP, генерация ресайзов под разные размеры экранов. Скрипт на sharp, минуты работы, заметное ускорение сайта.

**Текущее состояние:** частично закрыто — `scripts/process-images.mjs` уже делает webp + размеры для товаров. См. [assets-pipeline-extension.md](./assets-pipeline-extension.md) — там расширение на лого/hero/блог/story.

**AI-обвязка (идея заказчика):** Telegram-бот.
- Заказчица кидает в бот 1-3 фотки товара "как есть" (lifestyle, на манекене, на вешалке — что угодно).
- Бот выдаёт 3 маркетплейсные фотки **на белом фоне** (front/side/back).
- Под капотом — наш flat-shot pipeline (см. [flat-shots-pipeline.md](./flat-shots-pipeline.md), пункт 3).

**Связка с другими бэклог-задачами:**
- Базовая оптимизация уже частично есть → нужно довести через [assets-pipeline-extension](./assets-pipeline-extension.md).
- Бот — это фактически потребительский UI поверх [flat-shots-pipeline](./flat-shots-pipeline.md). Можно начать с TG-бота как MVP вместо веб-приложения (быстрее, не надо хостинг/auth/Stripe).

**Что обсудить:** делать TG-бот как личный инструмент для заказчицы (бесплатно для неё, мы тренируем pipeline) или сразу как продукт для других брендов?

## 2. Продуктовая фотосессия через Imagen / nano-banana

**Базовая ценность:** lifestyle-кадры товара в outdoor-сеттингах (горы, лес, кемпинг) без выезда на съёмку. У фотографа это 50-100 тыс тенге, через AI — час работы.

**AI-обвязка:** тут заказчик честно сказал "не очень шарю, тяжело работать с фотками в таком формате". Это значит **рисковая зона** — без насмотренности промптить AI-фотосессию даст шлак.

**Что нужно проработать прежде чем брать в работу:**
- Собрать референсы (Patagonia, Arc'teryx, Burton — реальные кадры, чтобы знать целевой look).
- Тест на одном товаре: можно ли через nano-banana / Imagen вставить **существующий товар** в outdoor-сеттинг (не сгенерировать новый товар, а взять flat-shot и поместить в горы) — это проверка реалистичности.
- Альтернатива: использовать lifestyle-кадры которые **уже есть** в `assets/lifestyle/` (110 файлов!) как базу и докручивать — может оказаться, что собственного материала достаточно.

**Гипотеза:** в чистом виде "сгенерируй красивого человека в Хан Шелл на фоне Алатау" не получится — модели плохо рендерят логотипы и точные цвета. Реалистичнее: **взять реальную фотку** + AI-композ фона / погоды / времени суток.

**Приоритет:** низкий пока не проверим гипотезу. Зафиксировать как research-задачу: 2-3 часа на тест → решение брать/не брать.

## 3. Telegram-бот для приёма заявок

**Базовая ценность:** форма "узнать о наличии" / "оформить заказ" с сайта → сообщение в Telegram заказчицы. 30 минут работы (Next.js API route + bot token).

**AI-обвязка (идея заказчика):** бот не просто пересылает заявку, а **продолжает диалог в чате на сайте**.
- Покупатель пишет "интересует Хан Шелл, есть ли красный M?"
- Бот отвечает (по `products.ts`) с наличием, ценой, ссылкой на товар
- Если вопрос вне базы — эскалирует в Telegram заказчицы с контекстом
- Заказчица отвечает в Telegram — ответ долетает обратно в чат на сайте

**Стек идеи:**
- Источник истины — `products.ts` (он у нас типизированный, легко скармливать LLM как контекст).
- LLM — Claude Haiku или GPT-4o-mini (дёшево, быстро, для простых вопросов хватит).
- Telegram bot API для эскалации + ответов от заказчицы.
- Чат-виджет на сайте — простой компонент, можно взять готовое (Crisp / Tawk / самопис).

**Сложности:**
- Связать "анонимный посетитель сайта" ↔ "Telegram-сессия заказчицы" через ID (cookie/localStorage + сообщение в TG с reply_to).
- Стейт диалогов — где хранить? Без БД будет либо in-memory (умирает с рестартом), либо на vercel KV / upstash.
- Безопасность — чтобы случайные люди не задосили API.

**Приоритет:** базовый бот = быстро (30 мин), AI-чат = отдельная фича на 1-2 дня. Можно делать поэтапно: сначала тупая форма → TG, потом обвешать AI.

## 4. Лендинг под рекламу для одной коллекции

**Базовая ценность:** отдельная страница "Зимняя коллекция" с UTM-метками, оптимизированная под таргет (Instagram, ВК). Цена услуги $500-800.

**AI-обвязка (идея заказчика):** агент в админке.
- Заказчица заходит в админку, говорит "сделай лендинг для зимней коллекции".
- Указывает товары (или агент сам выбирает по тегам/категории/сезону).
- Агент:
  - генерит копирайтинг (заголовки, описания, CTA),
  - подбирает фотки из каталога,
  - собирает лендинг (Next.js страница из шаблона),
  - параллельно генерит **посты для соцсетей** (Instagram-карусель, сторис, ВК-пост) с теми же визуалами и копирайтом.

**Стек идеи:**
- Шаблон лендинга — параметризованная Next.js-страница (hero, секция товаров, story-блок, CTA).
- Агент — Claude с тулзами:
  - `read_products` (читает `products.ts`)
  - `select_photos` (выбирает фото из `public/images/products/`)
  - `generate_landing_page` (заполняет шаблон → коммит в git → деплой)
  - `generate_social_posts` (генерит изображения через nano-banana + копирайт + раскладывает по форматам)
- Админка — простая форма "тема кампании / товары / даты" + чат с агентом.

**Это фактически продукт уровня "AI marketing copilot для маленького бренда".** Сложность высокая, но и ценность тоже — это можно перепродавать другим брендам.

**Сложности:**
- Авторизация админки (заказчица должна логиниться, никто другой).
- Деплой через агента — нужен safe-режим (preview deploy на vercel перед merge в main).
- Качество AI-копирайта на русском — надо тестировать. Для маленьких объёмов норм, для серьёзного маркетинга может потребоваться ручная редакция.

**Приоритет:** самое амбициозное из четырёх. Брать когда (a) заказчица увидит ценность v1 сайта и (b) появится бюджет на 1-2 недели работы.
