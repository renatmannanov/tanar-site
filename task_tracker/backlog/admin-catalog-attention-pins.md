# Бэклог: пины «требует внимания» в каталоге админки

> Источник: фидбек заказчика 2026-06-11 (после marketplace-sku-links).
> Статус: backlog, в работу не взято.

## Идея

Админ должен с первого взгляда видеть, какие товары недозаполнены или
распроданы — не открывая каждую карточку.

## Хотелки

1. **Пин на «Каталог» в сайдбаре** — как у «Заказов» (счётчик pending):
   количество SKU, требующих внимания (см. определение ниже).
2. **Список каталога: счётчик незаполненных полей** — справа от названия
   товара, сколько незаполненных полей у него (по товару + по его SKU).
3. **Список каталога: красный пин «остаток 0»** — справа, количество товаров
   (или SKU) с нулевым остатком. Отдельный от счётчика полей, красный.

## Что считается «незаполненным полем» (определение заказчика)

- Любое продуктовое поле: категория, статус, цена, описание, бейдж и т.д.
- Незаполненные поля в размерах (SKU: артикул, ruSize, остаток…).
- Отсутствие фото (у цвета нет ни одного media_asset).
- Отсутствие ссылки (marketplaces: продуктовые и/или sku-ссылки Ozon/Kaspi).

## Заметки для реализации (когда возьмём)

- Точный список полей и веса «важности» — уточнить у заказчика перед стартом:
  «любое поле» буквально = шум (например, пустой `care` у футболки — норм?).
  Возможно, два уровня: «критично» (цена 0, нет фото, нет ссылок, остаток 0)
  и «желательно» (бейдж, care, specs).
- Пин в сайдбаре у «Заказов» уже есть — переиспользовать его паттерн
  (server-компонент сайдбара считает count).
- Считать на SQL-уровне одним запросом (каталог маленький — можно и в JS
  поверх getAllProducts + listProductImagesForProducts).
- «Остаток 0» — по `available = stockQty − reservedQty` или по `stockQty`?
  Уточнить (для пина логичнее stockQty = 0).
